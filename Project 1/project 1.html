The basics of the Web and HTML

HTML is what glues everything together. Web has 30 billion pages. Major pieces of the web: you, your computer, the internet and servers HTTP-the main protocol of the web HTML: hypertext markup language. HTML is made up of text content, markup (what it looks like), references to other documents (video, images). Links to other documents. HTML is made up of tags. The first is the opening and the last is the closing element. b tag is for bold . You need an opening and closing tag. em is for emphasis. It italicizes the text.

HTML attributes: Anchor tag (allows you to create a link). Visit our HTML tutorial
Image tag "img" allows you to insert an image.
Break tag (br)-will insert a break

Paragraph-will also insert a break.

There are two types of elements: inline and block elements. Inline elements are: b, em, img, span, Block elements: p, div Div elements can allow you to break up content by placing them in a container. You can also accomplish this through span tags. Span tags can also be used to break up content within a paragraph. For example, I used a span tag here to change the work brown to brown.
HTML document-the head section contains meta data, java script, css Body: body tags has all the content in it, which is the stuff you see on the screen

Web Page Structure

The basic word in the language of html is tags. DOM is the: document object model. CSS defines the style of an object. Boxes, boxes everywhere: Everything on a web page is actually a box. Here are some main points:
  • 1. Draw boxes around your page
  • 2. Use Divs to create boxes
  • 3. To add style, you need to add classes to your boxes
  • The browser builds a tree like structure of the page.
The analogy of a tree is important to comprehend on how HTML works. Trees have many branches. The main branches of a tree can have their own branches and so on. In HTML you can have elements within elements. Using a tree structure is essential to building and organizing your page. Another analogy that can be used is boxes within boxes which is what you do when you build a web page.

Avoiding repetition-Adding CSS Style to HTML Structure

CSS allows you to avoid repetition and make good code. CSS stands for cascading style sheets. CSS works on the concept of inheritance. Wikipedia defines it as follows: Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. Inheritance relies on the document tree, which is the hierarchy of (X)HTML elements in a page based on nesting.

The first thing you do in CSS is to write a selector. You can apply styles to classes, paragraphs etc A good reference for CSS properties can be found at this Mozilla web page: Mozilla web page One other important thing: You have to refer to the CSS file in the header to make the page work. The header should refer to a separate CSS file. You can also include CSS in your html (this would create a lot of repetition) or put all the CSS in the header vs a separate sheet. Using semantic tags:You should use h1, h2, and h3 tags to describe what is in your page. Search engines also use these tags.
The box model: For a box you have margin, border, padding and content. Content is the content (like an image). Padding clears area around content. Border goes around padding and content. Margin clears area outside of border. Sizes of boxes should be set as percents instead of pixels. Divs naturally take up an entire row. If you want divs (boxes) stacked next to each other you can use flex box.
To add images: 1. Save your image to a separate folder. 2. To get your image to show up on the page you need to make sure the code is in your html. Make sure you use the appropriate code including img=src and then the folder you are storing it in.
